# TJA譜面エディタ

太鼓さん次郎系の `.tja` 譜面をブラウザ上で作成・編集できるWebアプリです。
テキストエディタでTJAを直接書けなくても、譜面Wikiのような横長の小節図をクリックで編集し、
音源と同期再生してズレを確認しながら `.tja` としてエクスポートできます。

すべてローカルブラウザ内で完結します（サーバー不要・音源はアップロードされません）。

4つのタブがあります。

- **✏ 編集** — 小節図でノーツを配置・編集する
- **📜 全体** — 譜面Wikiの譜面画像のように全小節をコンパクトに一覧表示（再生位置バーも流れる、クリックでジャンプ）
- **🥁 オート再生** — ノーツが流れてきて自動で叩かれ、右上へ飛んでいく本家風のプレビュー
- **📂 ファイル** — .tja / プロジェクトJSONのエクスポート・インポートと統計の確認

## 起動方法

```bash
npm install
npm run dev
```

表示されたURL（通常 http://localhost:5173）をブラウザで開いてください。

本番ビルドは `npm run build`（`dist/` に出力）です。

## 使い方

1. **曲情報を入力** — 画面上部でTITLE・BPM・COURSE・LEVELなどを入力します。
2. **音源を読み込む** — 「🎵 音源を読み込む」からローカルの mp3 / ogg / wav を選択します。
   ファイル名は自動で `WAVE:` に反映されます。
3. **ノーツを置く** — パレットでノーツ種別（ドン・カッ・大ドン…）を選び、小節図の線の上をクリック。
   ノーツはグリッド線の上に中心が乗り、太鼓の達人風に16分間隔から隣同士が重なって表示されます（先のノーツが上）。
   右クリックまたは消しゴムモードで消せます。
4. **入力単位を選ぶ** — パレット右端の「入力単位」で16分・24分などのスナップを切り替えます。
   TJAの分割数は配置済みノーツから**自動で最小に調整**されます（例: 16分と24分が混在→48分割、
   24分を消せば16分割に戻る）。手動で分割を管理する必要はありません。
5. **ノーツをまとめて消す** — レーン上をドラッグ（またはShift+クリック）で範囲選択し、
   Delete か「選択ノーツを削除」ボタンで削除します。小節をまたぐ選択もできます。
6. **小節を編集** — 小節ヘッダをクリックで選択すると、下に拍子・BPM変更・SCROLLの入力欄が出ます
   （非選択の小節は変更値だけヘッダに文字表示）。拍子を変えるとノーツの拍位置を保ったまま伸縮します。
   ヘッダのボタンで複製・空小節挿入（＋）・削除（✕）ができます。
7. **再生して確認** — ▶再生で音源とノーツ音（ドン・カッ）が鳴り、赤い縦棒が譜面上を移動します。
   連打・風船区間は自動で約20分音符間隔のドン連打が鳴ります（風船は必要打数で停止）。
8. **オフセットを合わせる** — ノーツ音と曲がズレていたら「+0.001 / -0.001 / ±0.01」ボタンで調整。
   OFFSETを増やす(+)と譜面が早く、減らす(-)と遅くなります。
   端末固有の遅延は下の「**再生環境補正**」で合わせてください（TJAには出力されず、この端末にのみ保存）。
9. **エクスポート** — 「📂 ファイル」タブの「⬇ .tja をダウンロード」で保存。プレビュー表示もできます。

### オート再生タブ

- 「▶ スタート」で助走付きのオート再生が始まり、判定枠に到達したノーツは自動で叩かれて右上へ飛んでいきます。
- Q（またはキャンバスをタップ）で一時停止/再開、一時停止中に ← → で小節移動できます。
- ノーツの流れる速さは各小節のBPMとSCROLL（HS）を反映します。

### 再生の仕組み（音ズレ対策）

再生開始のたびに、音源とすべてのノーツ音（連打の自動ドン含む）を同一のAudioContextクロックへ
サンプル精度で一括スケジュールする「理論ミックス」方式です。ミックスはブラウザのオーディオ
スレッドが行うため、スマホでもタイマーの揺れによる音ズレが起きません。OFFSETや環境補正を
再生中に変更すると、その場で現在位置から組み直されて即座に反映されます。

### キーボード操作

| キー | 動作 |
|---|---|
| D / Shift+D | ドン / 大ドン |
| K / Shift+K | カッ / 大カッ |
| R / Shift+R | 連打開始 / 大連打開始 |
| B | 風船開始 |
| E | 連打・風船終了 |
| Delete / Backspace | 選択ノーツ削除 / カーソル位置を空白に |
| 0〜8 | 数字で直接入力 |
| 矢印キー | カーソル移動（←→で入力単位1つ、↑↓で小節） |
| Esc | ノーツ範囲選択の解除 |
| Space | 再生 / 一時停止 |
| Ctrl+Z / Ctrl+Y | 元に戻す / やり直す |
| Ctrl+C / Ctrl+V | 選択小節のコピー / 貼り付け |

キー入力はカーソル線（小節図をクリックした位置）に配置され、自動で次の線へ進みます。
カーソルは線そのものを指します（配置される位置と一致）。

### 範囲選択

- **ノーツ範囲**: レーン上をドラッグ、またはShift+クリック。Deleteで削除。
- **小節範囲**: 小節ヘッダをクリックで選択、Shift+クリックで範囲。ツールバーからコピー・貼り付け・削除。

## 対応しているTJA仕様

- ヘッダ: `TITLE` `SUBTITLE` `BPM` `OFFSET` `WAVE` `COURSE` `LEVEL` `BALLOON` `SCOREMODE` `SCOREINIT`
  `SCOREDIFF` `SONGVOL` `SEVOL` `DEMOSTART`
- 譜面本体: `#START` / `#END`、小節ごとの数字列（`0`〜`8`）+ カンマ
- 小節先頭単位の `#BPMCHANGE <number>`、`#MEASURE <分子>/<分母>`、`#SCROLL <number>`
- `#GOGOSTART` / `#GOGOEND`（小節単位。編集画面ではレーンが赤く着色され、選択小節の
  「ゴーゴー」チェックで切り替え。プレイ画面でもレーンが赤くなります）
- **複数コース入りTJAのインポート**: コースごとに `#START`〜`#END` を繰り返すファイルを読み込むと、
  どのコースを開くか選択できます（編集・書き出しは1コースずつ）
- ノーツ: 0=空白 / 1=ドン / 2=カッ / 3=大ドン / 4=大カッ / 5=連打開始 / 6=大連打開始 / 7=風船開始 / 8=終了
- 連打・風船は開始〜終了(8)が帯で接続表示されます。風船の必要打数は BALLOON 欄で編集でき、
  風船ノーツの増減に合わせて個数は自動調整されます（不足分は5打）。
- バリデーション: BPM > 0、LEVEL範囲、連打・風船の開始/終了対応、BALLOON個数の一致、SCROLL > 0 など

### OFFSETの扱い

TJAの `OFFSET:` は「譜面1小節目が始まる音源内時刻を負にした値」です。
例えば `OFFSET:-1.5` は音源の1.5秒地点から1小節目が始まることを意味します。
アプリ内部の同期式は `音源時刻 = 譜面時刻 - (OFFSET + 再生環境補正)` です（`src/utils/timing.ts` 参照）。
再生環境補正は端末・ブラウザごとの再生遅延を打ち消すためのローカル設定で、TJAには出力されません。

## 未対応の仕様（今後の拡張ポイント）

- 譜面分岐（`#BRANCHSTART` など）
- `#DELAY`
- 小節の途中でのBPM・拍子・SCROLL・GOGO変更（インポート時は次の小節の先頭として解釈し警告を表示）
- 複数コースの同時編集・一括書き出し（読み込みはコース選択式、書き出しは現在のコースのみ）
- ダブルプレイ（`#START P1/P2`）

`Measure` 型（`src/types.ts`）にフィールドを追加していく前提のデータ構造にしてあります。

## iPhone・iPadでのアプリ風利用（PWA）

このサイトはPWA対応です。iOSのSafariで開き、共有ボタン →「**ホーム画面に追加**」すると、
Safariの枠なし（スタンドアロン表示）の独立アプリのように起動します。

- ホーム画面のアイコン・アプリ名は同梱のマニフェスト／`apple-touch-icon` が使われます
- Service Workerにより、一度開いた後はオフラインでも起動できます（本番ビルドのみ有効）
- ノッチ・ホームバーはセーフエリア対応済み。タッチでのノーツ配置・ドラッグ範囲選択にも対応

### モード選択とスマホ専用UI

起動時に「**PC版／スマホ版**」を選ぶ画面が表示されます（端末に合わせた「おすすめ」バッジ付き。
ヘッダの「📱スマホ版へ／💻PC版へ」ボタンでいつでも切り替え可能）。
この選択画面には広告用のプレースホルダ（`.ad-slot`、`src/components/ModeSelect.tsx`）があり、
公開時にAdSense等のコードへ差し替えられます。

スマホ版を選ぶと以下のUIになります。

- **編集**: 画面下部に固定の入力パッドが出ます。上段は「⏮ ◀ ▶ ⏭（カーソル移動）・入力単位・↩（Undo）・▶（再生/停止）」、
  下段はノーツボタン（押すとカーソル位置に置いて自動で次の線へ進む＝テンポ入力）と
  「空白（スキップ）・🗑（消去）・消（消しゴムモード）」。PCのパレットの代わりになります。
- **全体表示**: 小さめスケールで1行に多くの小節が入ります。
- **プレイ**: 画面タップで演奏できます。**中央タップ=ドン、左右端（1/4ずつ）タップ=カッ**
  （2本指同時押し対応）。ゾーンの区切りは開始前・一時停止中に表示されます。
  キー設定はスマホでは非表示になり、「◀小節 / 小節▶」ボタンで一時停止中の小節移動ができます。
- 編集データはlocalStorageに自動保存されます（iOSはサイトを長期間使わないと消えることがあるため、
  大事な譜面は「プロジェクトJSONを保存」か .tja エクスポートでバックアップ推奨）

**公開時の注意**: Service Worker・ホーム画面追加のフル機能には **HTTPS配信が必須**です
（Vercel / Netlify / Cloudflare Pages などは標準でHTTPS）。`npm run build` の `dist/` を
そのまま配置すれば動きます。サブディレクトリ配下に置く場合は `vite.config.ts` の `base` を設定してください。
アイコンを変えたいときは `node scripts/gen-icons.mjs` で再生成できます（依存ライブラリ不要）。

## データの保存

- 編集内容は自動でブラウザの localStorage に保存され、次回開いたとき復元されます。
- 「プロジェクトJSONを保存 / 読み込み」でファイルとしてのバックアップもできます。

## 素材について

- 同梱のノーツ音（`public/sounds/don.wav` / `ka.wav`）は汎用の効果音です。
- 公式ゲームの画像・音声・ロゴ等の著作物は使用していません。

## 主要ファイル構成

```
src/
  types.ts              型定義（Project / Metadata / Measure / NoteValue）
  project.ts            プロジェクト生成・小節複製・正規化
  noteInfo.ts           ノーツの表示情報
  utils/
    timing.ts           小節・スロットの時刻計算、OFFSET同期
    fraction.ts         小節内位置の既約分数演算
    noteOps.ts          線上配置・自動分割調整・範囲削除・拍子変換
    rolls.ts            連打/風船の区間計算（帯表示・自動連打音用）
    tja.ts              TJA生成・パース・ダウンロード
    validation.ts       バリデーション
    stats.ts            ノーツ数・密度などの統計
  hooks/
    useHistory.ts       Undo/Redo
    usePlayer.ts        音源＋ノーツ音の再生エンジン（Web Audio / HTMLAudio）
  components/
    Lane.tsx            1小節ぶんの譜面レーン描画（編集・全体表示で共用）
    EditView.tsx        編集タブ（小節一覧・ドラッグ選択）
    OverviewView.tsx    全体表示タブ
    PlayView.tsx        プレイタブ（canvas・判定・キー設定）
    ほか                メタデータ・音源・オフセット・パレット等
```
